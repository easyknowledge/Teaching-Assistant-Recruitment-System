\## 📖 About The Project



Traditional Teaching Assistant (TA) recruitment processes are often bogged down by scattered emails, opaque selection criteria, and inefficient communication. \*\*This project delivers a comprehensive system design and high-fidelity prototype\*\* aimed at digitizing and streamlining the entire TA recruitment lifecycle. 



Developed as the capstone group project for \*\*EBU6304\*\*, this repository encapsulates the complete Software Engineering lifecycle—from initial requirements gathering and Agile backlog planning to UI/UX prototyping and system architecture design.



\### ✨ Key Features



\* \*\*Role-Based Access Control:\*\* Distinct workflows for Applicants (Students), Module Leaders, and System Administrators.

\* \*\*Agile Requirements Engineering:\*\* Comprehensive Product Backlog defining user stories, acceptance criteria, and sprint priorities.

\* \*\*Interactive UI Prototype:\*\* High-fidelity HTML/CSS prototype demonstrating user journeys and system usability.

\* \*\*Robust System Architecture:\*\* Detailed UML diagrams and database schemas documented in our final engineering report.
